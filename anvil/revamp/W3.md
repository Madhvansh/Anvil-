# Wave 3 — Proven-Accuracy Actionable Engine (data-first → selective prediction → sized advice)

## Context

The owner's #1 goal: **sustained 70–80%+ directional hit-rate**, surfaced as **a lot of actionable, sized
investment advice**, so the owner can **maximize their own trading profit** from using the tool. "Maximum
monetization" = max P&L from *acting on the calls* — **not** selling the app (the app stays flat-free, no
paywalls/tiers). The current docs (`anvil/docs/ANVIL.md` §1) describe the product wrongly — "paper + analytics
only / no investment advice / no monetization / never BSM on spot" — and must be corrected.

**The honest mechanism for 70–80% (centerpiece).** Raw every-day up/down accuracy caps near ~53–57%
(`anvil/docs/hypothesis.md`, the Wave-2 red-team); any tool claiming 70–80% on *every* call is leakage/overfit
and would destroy the product's identity (*"it sells what you can audit … a live, public reliability curve"*).
The achievable, auditable route is **selective prediction**: the engine **abstains unless its calibrated
confidence is high**, so it is right **70–80%+ *when it speaks***, proven live on the reliability curve. It
still produces a steady stream of calls because it spans many underlyings × strikes × horizons × structural
targets (P-touch, VRP, regime, cross-sectional). The existing DSR/PBO/t≥3 validation gate stays — it is what
makes the number *real*. Two binding unlocks: (1) **data** — only ~3 months of history is cached today; the
gate needs ~12–24 months to ever light up; (2) **the accuracy/abstention engine** itself.

This wave (**W3**) fuses and supersedes the old roadmap's W2.5 (data) + W4 (ML meta-layer) and adds the
advice/monetization layer. Owner-confirmed knobs: accuracy-first (tunable threshold), personal-use (full
advice), sized tickets + money-meter (manual placement, no auto-exec), data-first sequencing.

## M2 — BSM-on-spot engine (owner's explicit Greeks correction)

- **New `anvil/engine/bsm_spot.py`** — Black-Scholes-Merton with continuous dividend yield `q` on **spot**:

  BSM(S,q); divergence beyond tolerance = data-quality flag *or* a basis/mispricing feature fed to M3.

## M3 — SELECTIVE-PREDICTION ACCURACY ENGINE (delivers the 70–80%)

The heart. Brought forward from W4 and innovated. All pure-numpy (LightGBM optional via its py3-none wheel).

- **Multi-target conviction ensemble** `anvil/engine/conviction_ensemble.py` — blend the existing calibrated
  structural reads into one directional/structural conviction per opportunity: P(touch)
  (`engine/touch_probability.py`), VRP-rich (`engine/realized_vol_forecast.py`), regime agreement
  (`engine/regime_score.py`), term/crush (`engine/term_structure.py`), RND directional
  (`engine/implied_dist.py`), cross-sectional equity momentum (`factors/equities.py`). These targets calibrate
  high naturally — this is where 70–80% lives.
- **Calibration map** `anvil/engine/calibration_map.py` — **isotonic regression (pure-numpy PAV)** + Platt
  fallback mapping raw scores → calibrated probabilities; fit on M1 history, refit nightly.
- **Conformal bands** `anvil/engine/conformal.py` — split / Mondrian (regime-conditioned) conformal →
  guaranteed-coverage prediction intervals + an act/abstain signal. Pure-numpy.
- **Selective-prediction gate** `anvil/tips/selective.py` — **fire only when** calibrated p ≥ threshold
  (tunable, default ~0.70) **AND** conformal band excludes the no-edge region **AND** decision-brief verdict ≠
  ABSTAIN. This is the owner's accuracy-first posture. **Log coverage** ("fired on X% of opportunities") so
  honesty is preserved. Threshold is a `confidence_threshold` setting (`anvil/config.py`) + UI slider.
- **LightGBM meta-layer (optional)** `anvil/engine/meta_label.py` — López de Prado **meta-labeling**: GBM over
  **path/dynamics features** (ΔIV-rank, ΔGEX, OI velocity, term-slope changes, RV trajectory) predicts P(base
  signal correct) → act/abstain; pure-numpy logistic fallback. Gated by the same DSR/PBO/t≥3 battery.
- **Validation wiring**: every fired call records its calibrated prob to the ledger; with M1 data the existing
  `anvil/backtest/aggregate.py` (`validate_cells`, `cell_from_daily`) + `anvil/tips/gate.py` (`decide_tier`)
  finally light up `headline_eligible` / "Edge-verified ✓". Reuse `anvil/backtest/validation.py`
  (`deflated_sharpe_ratio`, `probability_of_backtest_overfitting`, `combinatorial_purged_splits`) unchanged.
- **Acceptance**: on 24 months, the gated subset shows ≥70% calibrated hit-rate on resolved forecasts with
  n≥50/cell; reliability curve near-diagonal (ECE < 0.10) per `anvil/ledger/scoring.py`.

## M4 — ACTIONABLE SIZED ADVICE + MONEY-METER (monetization-through-use)

- **TradePlan generation** — extend `anvil/engine/decision_brief.py` + `anvil/strategy/generate.py` to emit, per
  FAVORABLE high-conviction opportunity, a concrete plan: direction (BUY/SELL CE/PE/future or spread), **Kelly-
  sized lots** (reuse `anvil/strategy/sizing.py` `size_units`/`kelly_fraction_star`), entry zone, stop, target,
  max-loss/max-profit ₹, horizon, one-line rationale.
- **Money-meter** `anvil/strategy/money_meter.py` — given operator capital + calibrated edge + Kelly sizing,
  project **expected ₹ profit, ₹ at risk, expected weekly/monthly P&L** if the calls are followed. Ties accuracy
  directly to money.
- **Backtest-to-PnL bridge** — run the validated signals through the existing `anvil/paper/` subsystem
  (`paper/costs.py` realistic India F&O costs) to produce a **"what you would have made" 2-year equity curve** —
  the headline proof the calls make money, not just hit.
- **Personal full-advice mode** — `PERSONAL_MODE` flag in `anvil/config.py` that relaxes
  `anvil/agent/guardrail.py` (`check_compliance`) so analyst + UI speak direct buy/sell/sized/target language;
  default ON for the operator's machine. Compliance-safe mode stays the code default (ADR 0006).
- **Acceptance**: every high-conviction call renders a full sized ticket + ₹ projection; the 2y paper equity
  curve is shown and positive.

## Risks & honest notes
- **70–80% is a *gated* hit-rate, not all-call.** Coverage is always logged; the reliability curve is the proof.
  Pushing accuracy onto every call = leakage = self-defeating.
- **Data fetch is best-effort** (NSE/BSE archives rate-limit/move). Fail loud, cache, reconcile vs `nse_eod`.
- **Guardrail relax is gated by `PERSONAL_MODE`** — personal use only; shared/public builds keep compliance.
- **DSR/PBO/t≥3 gate is unchanged** — if real edge exists, it lights up with M1 data; if not, ✓ stays dark
  (by design, honestly).
